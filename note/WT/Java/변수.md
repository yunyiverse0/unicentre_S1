# 📝 JavaScript 변수 선언: var, let, const 완벽 정리

자바스크립트에서 변수를 선언하는 세 가지 키워드 (`var`, `let`, `const`)의 핵심 차이점과 재선언/재할당 개념 정리 

---

## 1. 핵심 차이점 요약 테이블

| 특징 | `var` (ES5 이하) | `let` (ES6 도입) | `const` (ES6 도입) |
| :--- | :--- | :--- | :--- |
| **스코프 (유효 범위)** | 함수 스코프 (Function Scope) | **블록 스코프 (Block Scope)** | **블록 스코프 (Block Scope)** |
| **재선언** | 가능 (덮어쓰기 위험) | **불가능** (안전) | **불가능** (안전) |
| **재할당** | 가능 | 가능 | **불가능** (상수) |
| **호이스팅** | 발생 (초기화: `undefined`) | 발생 (초기화 안 됨, **TDZ** 발생) | 발생 (초기화 안 됨, **TDZ** 발생) |
| **권장 사용** | 사용 지양 | 값이 변경되어야 할 때 | **기본적으로 사용** |

---

## 2. 재선언 (Redeclaration)과 재할당 (Reassignment)

두 개념을 혼동하지 않도록 주의해야 합니다.

### 🔹 재선언 (Redeclaration)
> **비유:** 같은 이름의 상자를 **또다시 만드는 것**
> **정의:** 변수 선언 키워드(`var`, `let`, `const`)를 사용하여 **다시 선언**하는 행위.

```javascript
// var는 가능:
var name = "A";
var name = "B"; // 덮어씀 (문제 발생 가능)

// let/const는 불가능:
let age = 10;
// let age = 20; // SyntaxError 발생
🔹 재할당 (Reassignment)
비유: 이미 있는 상자 안의 내용물만 바꿈 정의: 변수 이름과 = 연산자를 사용하여 새로운 값을 대입하는 행위. (키워드 사용 안 함)

JavaScript

// var, let은 가능:
let count = 1;
count = 3; // ✅ 가능

// const는 불가능:
const PI = 3.14;
// PI = 3.14159; // ❌ TypeError 발생
3. 스코프 (Scope) 차이: 블록 vs. 함수
🔹 함수 스코프 (Function Scope) - var
var는 함수 {} 내에서만 지역 변수로 인정됩니다. if문, for문 같은 블록 안에서는 무시되어 밖으로 새어 나갑니다.

JavaScript

function funcScope() {
  if (true) {
    var x = "Hello";
  }
  console.log(x); // 'Hello' (if 블록 밖에서도 접근 가능)
}
🔹 블록 스코프 (Block Scope) - let, const
let과 const는 {} 중괄호로 묶인 모든 코드 블록 내에서만 유효합니다. 블록을 벗어나면 접근할 수 없습니다.

JavaScript

function blockScope() {
  if (true) {
    let y = "World";
  }
  // console.log(y); // ReferenceError (블록 밖에서 접근 불가)
}
4. 호이스팅 (Hoisting)과 TDZ
호이스팅은 코드가 실행되기 전에 선언문이 스코프의 최상단으로 끌어올려진 것처럼 동작하는 현상입니다.

🔹 var 호이스팅 (초기화됨)
var는 호이스팅될 때 자동으로 undefined로 초기화됩니다.

JavaScript

console.log(varTest); // undefined 출력
var varTest = "값";
🔹 let/const 호이스팅 (TDZ 발생)
let, const는 호이스팅되지만 초기화되지 않습니다. 선언문 이전에 변수에 접근하려고 하면 **TDZ (Temporal Dead Zone)**라는 영역에 의해 ReferenceError가 발생합니다.

JavaScript

// console.log(letTest); // ReferenceError 발생 (TDZ 때문)
let letTest = "값";
